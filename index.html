the grep keyword lets you search for patterns of characters and displays all lines!

						$ grep "married" bank-additional.csv | wc -l # | is the piping operator

The sed keyword lets you modify any part of a file (NaN or misplacement) for a file that needs to be preprocessed 
 
						$ sed "s/, ?,/,,/g" adult.data >  adult.data
This will sub an empty string for a question mark, and will show that instead of NaN when you import and use pandas!
The uniq keyword almost functions with the same level as a diff; unique patterns can be identified Unfortunately, uniq is also dumb; can only detect reps when they are ADJACENT

A little about shell arithmetic: it always assumed a number input is base-8 unless you specify the base as such: 
 
							$ # echo $(([BASE]#[NUMBER]))
							$ echo $((013))
							  11
							$ echo $((2#101011))
							  43
							$ echo $((16#3E))
							  62
CRON: the unix time schedule software 
 

Min  Hour Day  Mon  Weekday
*    *    *    *    *  command to be executed
┬    ┬    ┬    ┬    ┬
│    │    │    │    └─  Weekday  (0=Sun .. 6=Sat)
│    │    │    └──────  Month    (1..12)
│    │    └───────────  Day      (1..31)
│    └────────────────  Hour     (0..23)
└─────────────────────  Minute   (0..59)
and you can model it to any function that either you make or import!



some examples of cron include: 

							$ # Custom cron job
